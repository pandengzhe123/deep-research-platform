# 用户体验批判：从用户视角审视全部痛点

> 站在使用者角度，不是开发者角度。只说实话。

---

## 一、启动和上手

### 1. 三个终端才能启动

现状：分别开三个窗口跑 Agent、网关、前端，再加 Docker 跑 PostgreSQL。`start.bat` 是有了，但用户要打开四个东西。

用户想：**"我能不能双击一个图标就开始用？"**

### 2. 登录后是空的什么都没有

注册登录成功 → 页面空白 → 一个输入框。没教程、没引导、没"试试问这个"。新用户甚至不知道这是一个深入研究工具——他们会问"今天天气怎么样"然后得到一个 2,000 字的深度报告，浪费了 API 费用也只有困惑。

用户想：**"我现在能做什么？这东西能回答哪些问题？"**

---

## 二、研究过程中

### 3. 点击后就是黑盒等待

输入问题后看到"正在分析问题并搜索相关信息..."和一个跳动的点。接下来 1-3 分钟没有任何反馈。用户不知道是在工作还是卡死了。计时器只告诉等了多久，不告诉在做什么。

用户想：**"这玩意是不是坏了？"**

### 4. Level 选择无人解释

下拉框里有四个选项：极速、搜索反思、多路并行、Supervisor。名字很专业但普通用户听不懂。什么时候选哪个？选错了会怎样？没有人告诉他们。

用户想：**"有什么区别？我该选哪个？"**

### 5. 报告质量不稳定

同样的 Level 2、同样的模型，问"量子计算"和问"AI 发展趋势"出来的报告深度完全不同。没有任何质量预判——用户提交前不知道这次研究大概烧多少 token、要等多久、能拿到多深的报告。

用户想：**"我付了这次搜索的钱，值吗？"**

---

## 三、历史和数据管理

### 6. 会话列表是死的

侧边栏显示一堆会话截断到 15 个字。分不清"研究一下市场"和"研究一下市场！"——觉得它们是两个东西。不能重命名、不能搜索、不能删除、不能归档。三个会话之后就开始找不到了。

用户想：**"上周我搜的那个关于光伏的报告去哪了？"**

### 7. KB 面板太简陋

上传文件后只显示文件名。点不了预览、看不到文件里有什么、不知道 Agent 搜到了哪些片段。如果上传了三个相似的 PDF，自己都忘了哪个是哪个。

用户想：**"我上传的那个文件还在吗？Agent 有没有用到它？"**

### 8. 报告无法导出

一份漂亮的深度报告只能在浏览器里看。不能存成 PDF，不能复制成 Markdown，不能分享给别人。刷新页面后要重新点开会话才能看到。

用户想：**"我怎么发给同事？"**

---

## 四、错误和异常

### 9. 错误提示毫无帮助

"请求失败: 500 Internal Server Error"——用户不知道发生了什么。是我的网络问题？是 API 挂了？是问题太复杂了？应该重试还是算了？没有任何指引。

用户想：**"怎么办？"**

### 10. 没有网络断开恢复

如果在研究期间网络断了，用户完全不知道。计时器还在走，看起来一切正常。直到超时报错，已经等了好几分钟。

用户想：**"到底怎么了？"**

---

## 五、移动和跨设备

### 11. 手机打开体验很差

手机打开勉强能看到聊天，但 KB 面板、侧边栏、Level 选择全部挤压在一起。不想用了。

用户想：**"这个不是给手机用的吧。"**

### 12. 换设备没法继续

在办公室电脑上做了一个研究，回家想看一眼——看不到。因为聊天记录存在浏览器的 localStorage 里。登录了同一个账号但一台电脑看不到另一台的会话。数据库里有但前端不会自动发现新的。

用户想：**"我不是登录了吗？怎么看不到之前的研究？"**

## 七、结论

当前项目的**核心功能完整**——Agent 能搜、能写报告、能追问、能 RAG。但从"能用"到"好用"，差在六个字：**反馈、引导、导出**。

- **反馈**：进度不可见、错误无指引 = P0
- **引导**：新用户不知道做什么 = P1
- **导出**：成果无法带走 = P1

修 P0 和 P1 总共不到 100 行代码，但用户体验提升一个数量级。
